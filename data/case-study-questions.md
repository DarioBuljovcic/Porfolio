# Case Study Questions

Use this for each project (`arhangelapp`, `zgradar`, `inhaus`) to fill real data in `data/projects.ts`.

## 1) Basic Identity
- What is the exact project title?
- Write one line: what it does (not what stack it uses).
- Is this client work, personal project, or team project?
- If team project: how many people were on the team?

## 2) Problem It Solves
- What business/user problem existed before this project?
- What was failing in the old flow/process?
- Why was solving this problem important now?
- Who are the primary users?

## 3) Your Role
- What did you personally own end-to-end?
- Which parts did you not do?
- Did you make product decisions, design decisions, or only implementation decisions?
- Did you handle deployment/release? If yes, what exactly?

## 4) Key Decisions (Most Important Section)
- What were 3 key decisions you made?
- For each decision:
  - What were the options?
  - Why did you choose this option?
  - What tradeoff did you accept?
- Which decision had the biggest impact on quality/speed/business results?

## 5) Result / Outcome (Use Real Numbers)
- What measurable result improved after launch?
- Metrics to include where possible:
  - conversion rate
  - number of leads/inquiries
  - bounce/drop-off
  - page speed / Core Web Vitals
  - task completion time
  - support tickets / user complaints
- If you do not have hard metrics, what strongest qualitative signal do you have?

## 6) Links
- Live URL (final)
- GitHub URL (if public)
- If private code: one sentence explaining why

## 7) Optional But Strong
- Best screenshot for this project?
- Do you have a short demo video (30-90s)?
- Stack list (max 4-6 technologies you actually used)

## 8) 30-Second Hiring Manager Test
- Can someone understand in under 30 seconds:
  - what you built
  - why it matters
  - what you specifically contributed
- If not, what can be cut or rewritten?
